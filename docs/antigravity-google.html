Guide · Antigravity

Google Antigravity — Khái niệm, cách sử dụng và ứng dụng thực tế

Nguồn chính: antigravity.google/docs/home. Nội dung này được viết lại thành tài liệu đọc nhanh bằng tiếng Việt để lưu trong GitHub Pages.

1. Google Antigravity là gì?

Google Antigravity là nền tảng phát triển phần mềm theo hướng agent-first của Google. Thay vì AI chỉ gợi ý code từng dòng trong editor, Antigravity cho phép bạn giao một nhiệm vụ lớn cho AI Agent. Agent có thể lập kế hoạch, đọc code, sửa file, chạy terminal, kiểm thử, mở browser, tạo artifact và báo cáo lại kết quả.

Antigravity không chỉ là IDE có chatbot. Nó là môi trường làm việc nơi AI Agent được dùng như một “nhân sự kỹ thuật ảo” có thể thực hiện nhiều bước trong quy trình phát triển phần mềm.

Tư duy cốt lõi

IDE truyền thống:
User viết code -> AI gợi ý -> User tự kiểm tra -> User tự chạy test

Antigravity:
User giao mục tiêu -> Agent lập plan -> Agent sửa code -> Agent chạy test -> Agent tạo artifact -> User review

Điểm khác biệt lớn

Tiêu chí IDE AI thông thường Google Antigravity
Vai trò AI Trợ lý gợi ý code Agent có thể thực hiện task nhiều bước

2. Các khái niệm quan trọng

2.1 Agent

Agent là AI worker có khả năng nhận mục tiêu, chia nhỏ công việc, thao tác trên project và tạo kết quả.

  • Đọc code trong workspace.
  • Sửa file.
  • Chạy lệnh terminal.
  • Mở browser để kiểm tra UI/web app.
  • Tạo kế hoạch thực hiện.
  • Sinh artifact để người dùng review.
Phân tích toàn bộ repo Next.js này, tìm nguyên nhân trang dashboard load chậm, tối ưu lại component/table, chạy test và báo cáo thay đổi.

2.2 Agent Manager

Agent Manager là khu vực quản lý các agent và task. Có thể hiểu như “mission control” cho nhiều agent.

  • Theo dõi agent đang làm gì.
  • Chạy nhiều task song song.
  • Giao task cho từng workspace/project.
  • Review tiến độ, artifact, plan, kết quả.

2.3 Editor

Editor là giao diện viết code quen thuộc, gần giống VS Code. Đây là nơi bạn mở project, xem file, sửa file, review diff và chat với agent theo ngữ cảnh code.

2.4 Terminal

Agent có thể dùng terminal để cài package, chạy dev server, test, build, lint hoặc chạy script phân tích.

npm install
npm run lint
npm run test
npm run build

Terminal mạnh nhưng cần kiểm soát kỹ vì có thể thay đổi file, xóa file, ghi đè dữ liệu hoặc gọi network.

2.5 Browser

Browser tích hợp giúp agent kiểm tra app như người dùng thật: mở localhost, click UI, kiểm tra layout, chụp screenshot hoặc ghi lại browser recording.

2.6 Artifact

Artifact là sản phẩm trung gian hoặc kết quả do agent tạo ra để bạn kiểm chứng.

  • Task list.
  • Implementation plan.
  • Visual code diff.
  • Screenshot.
  • Browser recording.
  • Báo cáo kiểm thử.

2.7 MCP — Model Context Protocol

MCP là cơ chế kết nối Antigravity với tool/server bên ngoài như Supabase, GitHub, database tool, filesystem tool, internal API hoặc custom server riêng.

Agent đọc schema Supabase qua MCP, kiểm tra migration, sau đó sửa Prisma schema và cập nhật API route.

2.8 Skills

Skills là các gói hướng dẫn/tư duy/workflow có thể tái sử dụng cho agent.

skills/design-taste-frontend/SKILL.md

Dùng khi bạn có task lặp lại nhiều lần như thiết kế UI dashboard, dịch truyện, audit security, viết test, refactor Next.js hoặc review performance.

2.9 Rules / Workflows

Rules là luật hành vi áp dụng cho agent. Nếu skill là năng lực theo nhiệm vụ, rules là luật nền agent phải tuân thủ.

- Luôn trả lời bằng tiếng Việt.
- Không đổi public API nếu user không yêu cầu.
- Trước khi sửa code phải đọc file liên quan.
- Sau khi sửa phải chạy lint/build nếu project có script.
- Không tự ý xóa file dữ liệu.

2.10 Plugins

Plugins là gói mở rộng có thể gom nhiều thành phần như Skills, Rules, MCP servers, Hooks và Agents vào một package.

2.11 Hooks

Hooks là điểm móc để tự động chạy hành động ở một thời điểm nhất định: trước task, sau khi sửa code, trước khi hoàn tất hoặc sau khi tạo artifact.

2.12 Worktree Support

Worktree giúp agent xử lý task ở workspace riêng, giảm rủi ro làm bẩn branch chính và dễ chạy nhiều task song song.

main branch sạch
    ├── worktree/feature-dashboard-ui
    ├── worktree/fix-api-timeout
    └── worktree/add-tests

2.13 Antigravity CLI

Antigravity CLI là giao diện dòng lệnh cho agent, phù hợp khi muốn giao task từ terminal hoặc tích hợp vào workflow dev hiện tại.

3. Cách sử dụng Google Antigravity

3.1 Cài đặt

  1. Truy cập trang chính thức: antigravity.google.
  2. Tải bản phù hợp với macOS, Windows hoặc Linux.
  3. Cài đặt app.
  4. Đăng nhập bằng tài khoản Google được hỗ trợ.
  5. Mở project code local.
  6. Cấu hình model, rules, skills, MCP nếu cần.

3.2 Mở project

File -> Open Folder -> chọn project

Sau khi mở project, agent sẽ có context từ file, cấu trúc thư mục, Git diff, terminal output, rules và skills nếu đã cấu hình.

3.3 Giao task cho agent

Task nên viết rõ mục tiêu, phạm vi, ràng buộc và tiêu chí hoàn thành.

## Mục tiêu
[Việc cần làm]

## Phạm vi
[File/folder/page/module liên quan]

## Ràng buộc
[Không được đổi gì, giữ API nào, style nào]

## Tiêu chí hoàn thành
- Build pass
- Test pass
- UI không vỡ responsive
- Có summary thay đổi

3.4 Review plan trước khi sửa sâu

Đọc project và lập plan trước, chưa sửa code. Sau khi tôi duyệt mới triển khai.

3.5 Review diff và artifact

  1. Xem diff từng file.
  2. Đọc summary thay đổi.
  3. Kiểm tra terminal output.
  4. Xem screenshot/browser recording nếu có UI.
  5. Tự chạy lại build/test nếu task quan trọng.
  6. Commit thay đổi khi đã chắc.

3.6 Cấu hình Skills

.agents/
└── skills/
    └── ten-skill/
        └── SKILL.md

3.7 Cấu hình Rules

# Project Rules

- Luôn dùng TypeScript strict.
- Không dùng any nếu có thể định nghĩa type.
- Component UI phải responsive.
- Không đổi route public nếu user không yêu cầu.
- Sau khi sửa code phải chạy npm run lint và npm run build nếu có script.
- Không tự ý xóa file .env, database, migration hoặc dữ liệu user.

3.8 Dùng MCP

  1. Chọn MCP server phù hợp.
  2. Cấu hình server theo docs.
  3. Cấp quyền tối thiểu cần thiết.
  4. Test kết nối.
  5. Giao task có nêu rõ phạm vi MCP.

4. Ứng dụng thực tế

  • Build feature mới: thêm chức năng export CSV, search, filter, loading state.
  • Refactor codebase: tách service, repository, validation nhưng giữ API public.
  • Debug production: đọc log, tìm endpoint lỗi, đề xuất fix an toàn.
  • Tối ưu UI/UX: redesign dashboard, tối ưu table, responsive, empty state.
  • Viết test: unit test, integration test, edge case, error case.
  • Review bảo mật: kiểm tra upload, auth, path traversal, size limit.

5. Prompt mẫu dùng với Antigravity

5.1 Phân tích repo

Bạn là senior software engineer. Hãy phân tích repo này.

Yêu cầu:
- Giải thích kiến trúc tổng thể.
- Chỉ ra entrypoint chính.
- Liệt kê module quan trọng.
- Chỉ ra luồng dữ liệu chính.
- Chỉ ra rủi ro kỹ thuật.
- Chưa sửa code.

5.2 Fix bug

Mục tiêu: Fix lỗi [mô tả lỗi].

Context:
- Lỗi xảy ra ở: [page/API/module]
- Log: [paste log]
- Kỳ vọng: [hành vi đúng]

Ràng buộc:
- Không đổi API public.
- Không xóa test hiện có.
- Không sửa ngoài phạm vi nếu không cần.

5.3 Redesign UI

Dùng skill design-taste-frontend để redesign màn hình này.

Mục tiêu:
- Nâng cấp UI lên mức SaaS professional.
- Giữ nguyên logic và dữ liệu.
- Tối ưu table, card, spacing, typography.
- Có responsive mobile/tablet/desktop.

6. Best practices

  • Giao task rõ ràng, tránh prompt mơ hồ.
  • Chia nhỏ task lớn thành nhiều bước.
  • Yêu cầu plan trước với task rủi ro.
  • Luôn chạy lint, test, build nếu project có script.
  • Kiểm soát quyền terminal và MCP.
  • Review diff thủ công trước khi merge.

7. Rủi ro và lưu ý bảo mật

Antigravity mạnh vì agent có quyền thao tác thật. Nhưng quyền càng cao thì rủi ro càng lớn.

  • Agent sửa quá phạm vi yêu cầu.
  • Agent chạy lệnh terminal nguy hiểm.
  • Agent đọc nhầm hoặc ghi đè file quan trọng.
  • Prompt injection từ file Markdown, issue, webpage hoặc docs lạ.
  • MCP server cấp quyền quá rộng.
  • Thay đổi database/migration ngoài ý muốn.

8. Workflow khuyến nghị

Bước 1: Tạo branch/worktree mới
Bước 2: Giao task rõ ràng cho agent
Bước 3: Agent đọc code và tạo plan
Bước 4: User duyệt plan
Bước 5: Agent sửa code
Bước 6: Agent chạy lint/test/build
Bước 7: Agent tạo artifact/screenshot/report
Bước 8: User review diff
Bước 9: User tự test lại flow quan trọng
Bước 10: Commit/merge

9. Khi nào nên dùng Antigravity?

Nên dùng khi project có nhiều file cần đọc hiểu, task cần nhiều bước, cần agent kiểm tra UI bằng browser, cần chạy song song nhiều task hoặc cần chuẩn hóa workflow bằng skills/rules/hooks.

Không nên dùng bừa khi repo chứa secret, chưa có Git backup, task liên quan production database hoặc bạn không có thời gian review diff.

10. Tóm tắt nhanh

Google Antigravity là nền tảng IDE/agentic development của Google, giúp bạn giao việc cho AI Agent thực hiện các task phần mềm phức tạp. Cách dùng hiệu quả là viết task rõ, dùng plan trước khi sửa lớn, dùng skills/rules, kiểm soát MCP và review artifact/diff/test trước khi merge.
