Google Antigravity — Khái niệm, cách sử dụng và ứng dụng thực tế

Tài liệu này tóm tắt Google Antigravity theo hướng thực chiến: hiểu công cụ, biết cách giao việc cho agent, tổ chức workflow an toàn và ứng dụng vào dự án thật.

1. Google Antigravity là gì?

Google Antigravity là môi trường phát triển phần mềm theo hướng agent-first. Thay vì chỉ gợi ý code từng dòng, agent có thể nhận mục tiêu, đọc codebase, lập kế hoạch, sửa file, chạy terminal, kiểm tra browser, tạo artifact và báo cáo kết quả. 

IDE truyền thống:
User viết code -> AI gợi ý -> User tự kiểm tra -> User tự chạy test

Antigravity:
User giao mục tiêu -> Agent lập plan -> Agent sửa code -> Agent chạy test -> Agent tạo artifact -> User review

2. Các khái niệm chính

Khái niệm Vai trò
Agent AI worker có thể xử lý task nhiều bước trong codebase.
Agent Manager Khu vực quản lý task, tiến độ, artifact và nhiều agent song song.
Editor Nơi đọc code, sửa file, review diff và tương tác theo context project.
Terminal Dùng để cài package, chạy lint, test, build hoặc script.
Browser Kiểm tra UI/web app như người dùng thật, có thể ghi lại kết quả.
Artifact Bằng chứng để review: plan, diff, screenshot, recording, test output.
MCP Kết nối agent với tool/server bên ngoài như GitHub, Supabase, database, API nội bộ.
Skills Gói hướng dẫn tái sử dụng cho một loại task như UI design, refactor, test, audit.
Rules Luật nền buộc agent tuân thủ trong toàn project.
Plugins Đóng gói skills, rules, MCP, hooks và agent config thành một bộ cài.

3. Cách sử dụng cơ bản

  1. Cài Antigravity từ trang chính thức.
  2. Mở folder project local.
  3. Cấu hình rules, skills hoặc MCP nếu cần.
  4. Giao task rõ mục tiêu, phạm vi, ràng buộc và tiêu chí hoàn thành.
  5. Yêu cầu agent lập plan trước nếu task lớn.
  6. Review diff, artifact, screenshot và test output trước khi merge.

Prompt giao task chuẩn

## Mục tiêu
Tối ưu trang dashboard users để load nhanh hơn và UI chuyên nghiệp hơn.

## Phạm vi
- app/dashboard/users
- components/user-table
- API route liên quan nếu cần

## Ràng buộc
- Không đổi database schema.
- Không đổi response API hiện tại.
- Không xóa chức năng cũ.

## Tiêu chí hoàn thành
- npm run build pass.
- Table không overflow trên mobile.
- Có loading state và empty state.
- Trả về summary các file đã sửa.

4. Ứng dụng thực tế

  • Build feature: thêm export CSV, search, filter, loading/error state.
  • Refactor: tách service, repository, validation nhưng giữ API public.
  • Debug: đọc log, tìm endpoint lỗi, đề xuất fix an toàn.
  • UI/UX: redesign dashboard, tối ưu table, responsive, empty state.
  • Testing: viết unit test, integration test, edge case và error case.
  • Security audit: kiểm tra upload, auth, permission, path traversal, input validation.

5. Workflow an toàn

Bước 1: Tạo branch/worktree mới
Bước 2: Giao task rõ ràng cho agent
Bước 3: Agent đọc code và tạo plan
Bước 4: User duyệt plan
Bước 5: Agent sửa code
Bước 6: Agent chạy lint/test/build
Bước 7: Agent tạo artifact/screenshot/report
Bước 8: User review diff
Bước 9: User tự test lại flow quan trọng
Bước 10: Commit/merge

6. Best practices

  • Giao task rõ ràng, không dùng prompt mơ hồ kiểu “fix app này”.
  • Chia task lớn thành nhiều task nhỏ.
  • Luôn yêu cầu plan trước khi refactor lớn hoặc thay đổi kiến trúc.
  • Không cho agent tự chạy lệnh nguy hiểm như rm -rf, git reset --hard, prisma migrate reset nếu chưa duyệt.
  • Review diff thủ công, không merge chỉ vì agent báo done.
  • Giới hạn quyền MCP, nhất là với database hoặc production service.
